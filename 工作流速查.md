# 工作流速查（开赛后第一天打开本文件）

> 把本目录拷贝改名为新比赛名后，按以下顺序推进。每一步括号内是预计耗时与负责人。

## D0 — 开赛前 30 分钟

- [ ] **全员**：拷贝模板目录、`git init` 并推送到远端
- [ ] **全员**：检查 LaTeX 环境（`cd paper && latexmk main.tex` 跑通空摘要也算成功）
- [ ] **代码手**：检查 Python 环境（`pip install -r code/requirements.txt`）

## D0 第 1 小时 — 题目对齐

- [ ] **全员一起读题 30 分钟**，列出：
  - 输入数据是什么、规模多大
  - 输出是什么、判分依据
  - 哪些是模型选择题，哪些是开放题
- [ ] 把赛题 PDF 与附件放入 `problem/`
- [ ] **建模手**：起草 `model/模型定位说明.md` 第 1 节（总体链路图）

## D1 — 各角色起步

| 角色 | 起步任务 |
|------|---------|
| 建模手 | 填 `model/assumptions.md`（符号表 + 假设清单） + `model/参数总表.md` |
| 代码手 | 填 `code/src/data/pipeline.py`，跑出第一份 `code/data/processed/*.csv` |
| 论文手 | 填 `paper/sections/abstract.tex` 顶部参赛信息栏 + `1_introduction.tex` 背景段 |

## D1 末 — 第一次同步

- [ ] 建模手与代码手对齐符号命名（必要时改 `code/src/data/constants.py`）
- [ ] 代码手交付第一张数据概览图，论文手放入 `1_introduction.tex`

## D2 — 三问并行起跑

每问按 **建模手草稿（`formulation/Q*_模型.md`）→ 代码手实现（`code/src/q*/`）→ 论文手整合（`3_model.tex` / `4_solution.tex`）** 流水推进。

## D3 — 检验与敏感性

- [ ] 建模手细化 `model/敏感性分析设计.md` 与 `model/边界检验设计.md`
- [ ] 代码手按设计实现 `code/src/sensitivity/` 与 `code/tests/`
- [ ] 论文手填 `5_results.tex` 与 `6_sensitivity.tex`

## D4 — 摘要与一致性核查

- [ ] 论文手主笔 `abstract.tex` 摘要正文
- [ ] **三人共同**对照 `paper/一致性核查表.md` 逐项勾选
- [ ] 修正所有口径不一致项
- [ ] 编译 PDF，检查页数、图表清晰度、参考文献编号连续

## D-Final — 提交

- [ ] 按赛事要求重命名 PDF
- [ ] 检查参赛信息栏（编号 / 选题 / 赛道）已替换
- [ ] 备份完整工作目录（含 `code/data/processed/` 与 `code/results/`）
- [ ] 关停所有未保存的 Jupyter / 编辑器
- [ ] 提交！

---

## 出现混乱时的兜底原则

- **数据口径冲突**：以 `model/assumptions.md` 为准
- **参数数值冲突**：以 `model/参数总表.md` 为准
- **结果数值冲突**：以 `code/results/tables/` 中最新一次运行为准
- **图表版本冲突**：删 `paper/figures/` 重新从 `code/results/figures/` 复制
- **LaTeX 编译失败**：`latexmk -C` 清干净后从头编译
