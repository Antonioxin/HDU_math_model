## 问题 1 🔴：模型 A/B RUL 偏差 96%（1548d vs 57.5d）

### 根因
我原 Q1 §3.3 设模型 B 为"对数化电流增量的常漂移 Wiener"：$X=\ln(i-i_0+\epsilon)$。对指数退化 $i-i_0=\alpha(e^{\beta t}-1)$，$t\to0$ 时 $i-i_0\to0$、$\ln\to-\infty$ 存在**对数奇点**。端点法把 $X(0)=\ln(\epsilon)=-13.8$ 的人为跳变当成漂移，估出 $\mu=0.0076\approx21\beta$、$\sigma^2=1.447$ 被早期跳变灌爆 → RUL 低估到 57.5d。**这是模型设定错误，不是你的实现 bug**。

### 新公式：指数 / Λ-时间 Wiener 退化过程（Q1 §3.3 已重写）
弃用对数变换与 $\epsilon$。退化信号 $D(t)=i(t)-i_0$；运行时间 $\Lambda(t)=\alpha(e^{\beta t}-1)$（$(\alpha,\beta)$ **复用模型 A 的 NLLS 拟合**）：

$$D(t)=\Lambda(t)+\sigma_B B(\Lambda(t)),\quad \hat\sigma_B^2=\frac{1}{K-1}\sum_k\frac{(\Delta r_k)^2}{\Delta\Lambda_k},\ r_k=D_k-\Lambda(t_k)$$

RUL 首达时（运行时间内 IG，再经 $\Lambda^{-1}$ 映回日历）：
$$\Delta U\sim\text{IG}\Bigl(D_F-D_c,\tfrac{(D_F-D_c)^2}{\hat\sigma_B^2}\Bigr),\quad \text{RUL}=\tfrac1\beta\ln\Bigl(1+\tfrac{\Lambda(t_c)+\Delta U}{\alpha}\Bigr)-t_c$$

**点估计结构性 = 模型 A**：$\mathbb E[\Delta U]=D_F-D_c$，$\Lambda(t_c)\approx D_c$ ⇒ $s_{\text{eol}}\approx D_F$ ⇒ $\text{RUL}_B\approx\frac1\beta\ln(1+D_F/\alpha)-t_c=\text{RUL}_A$。

### 代码改动点（`code/src/q1/run_q1.py`）
- **删除** `_to_X()`（L114-115，对数变换）及所有 `EPS` 用法；
- **重写** `fit_wiener_model(...)`：签名改为 `fit_wiener(t, i, alpha, beta)`；内部算残差 `r = (i - I_0) - Lambda(t, alpha, beta)`、`dLam = np.diff(Lambda(t))`、`sigma_B2 = np.sum(np.diff(r)**2/dLam)/(K-1)`；返回 `{alpha, beta, sigma_B2}`；
- **重写** `rul_invgauss(...)` → `rul_lambda_wiener(alpha, beta, sigma_B2, i_c, t_c, i_F)`：
  ```python
  Lam = lambda tt: alpha*(np.expm1(beta*tt))
  Lam_inv = lambda s: np.log1p(s/alpha)/beta
  Dc, DF = i_c - I_0, i_F - I_0
  m_ig, lam_ig = DF - Dc, (DF - Dc)**2 / sigma_B2   # IG(mean, shape) on ΔU
  dU_med = m_ig                                      # 点估计
  t_eol = Lam_inv(Lam(t_c) + dU_med);  RUL = t_eol - t_c
  # CI: 对 ΔU 取 invgauss.ppf(0.025/0.975, ...) 再 Lam_inv 映射
  ```
- 失效阈用 `D_F = I_F - I_0`（**非**对数空间 `X_F`）。

### 重跑预期
`RUL_B ≈ RUL_A ≈ 1548 d`（附件2）；A/B 偏差从 96% 降到 **<2%**（仅来自 $\Lambda(t_c)$ 与 $D_c$ 的拟合残差）；`sigma_B2` 量级 $\sim\sigma_\varepsilon^2\approx 6\times10^{-5}$（不再是 1.447）。

## 问题 4 🔴：PHM Score 溢出 $3.67\times10^{54}$

### 根因
衍生自问题 1：旧模型 B 在附件1 中间时刻（t=1000/1500/2000）把 RUL 由 ~2500d 误估为 ~50d，$\Delta\approx-2450$，$\exp(2450/13)$ 溢出。叠加我 Q3 §4.5 Score 公式**无数值护栏**。

### 新公式（Q3 §4.5 已补护栏）
$$\text{Score}(\Delta)=\min\bigl(s(\Delta),\,S_{\max}\bigr),\quad S_{\max}=10^6$$

### 代码改动点（`code/src/q3/run_q3.py::phm_score`，L105-110）
```python
def phm_score(delta):
    d = np.clip(delta, -130, 130)                      # 防 exp 溢出
    s = np.where(d < 0, np.expm1(-d/13), np.expm1(d/10))
    return float(np.mean(np.minimum(s, 1e6)))          # S_max 护栏
```

### 重跑预期
修问题 1 后中间时刻 $\Delta$ 量级正常（数十天），Score 落在 $O(1)\sim O(10)$；护栏仅作冗余保险，正常不触发。

## 问题 2 ⚠️：迁移被全拦截（λ=0），最佳源 Bearing3_5 MMD=0.248

### 根因
你定位准确：[run_q3.py:117](../code/src/q3/run_q3.py#L117) `normalize_to_tau` **只归一化时间轴未归一化数值轴**。源 HI∈[0,1]，目标 $(i-i_0)/\Delta i_F$ 在附件1 末端达 1.082，高斯核 MMD 对该支撑域偏移过敏。叠加我原阈值 MMD_THR=0.10 过严。

### 新方案（Q3 §3.1/§3.2.3 已改）
1. **数值轴 min-max 归一化**：算 MMD/CORAL 前，源、目标曲线各自 min-max 到 [0,1]；
2. **联合判据，CORAL 主 + MMD 辅**：
   $$\text{transferable}\iff(\text{CORAL}<10^{-3})\wedge(\text{MMD}_{\text{norm}}<0.30)$$
3. 权重 $\lambda=\exp(-\text{MMD}_{\text{norm}}^2/h)$。

### 代码改动点（`code/src/q3/run_q3.py`）
- `normalize_to_tau(...)` 后**追加数值轴归一化**，或新增 `minmax_normalize(g)`，在 L243/L221 处对 `source_hi` 与 `g_target_att1` 都套一层；
- `compute_mmd` 入参改为归一化后曲线（命名 `mmd_norm`）；
- 新增 `is_transferable(coral, mmd_norm) -> bool` 实现联合判据；L259 `transferable = is_transferable(...)`；
- `MMD_THR` 0.10→0.30，新增 `CORAL_THR=1e-3`（见常量清单）。

### 重跑预期
Bearing3_5 数值轴归一化后 MMD$_{\text{norm}}$ 预计 ≪0.30，且 CORAL=3.8e-4<1e-3 ⇒ `transferable=True`、$\lambda>0$ ⇒ 迁移生效；Q3 RUL 的 95% CI 应**窄于** Q1（迁移收益）。

> **注意**：随 Q1 模型 B 重设，Q3 §4 迁移融合对象由旧 $(\mu,\sigma^2)$ 改为**退化曲率** $\kappa=\beta\cdot T_{\text{EOL}}$。详见 Q3 §4.1-4.4，代码接口见 §4 末"与代码的对应"。融合：$\hat\kappa_T^{\text{TL}}=\lambda\kappa_T^{\text{L2}}+(1-\lambda)\kappa_T^{\text{local}}$ → $\hat\beta_T^{\text{TL}}=\hat\kappa_T^{\text{TL}}/\hat T_{\text{EOL},T}$ → 代入 Λ-时间 IG。

## 问题 3 ⚠️(P2)：9 轴承终点全 F，过渡不清

### 根因
我 Q2 §3.6：固定 `PELT pen=10` 对长序列（Bearing3_1 有 2538 点）变点过少；`mean+3σ` 报警线受早期尖峰抬高。

### 新方案（Q2 §3.6 已改）
1. **稳健化报警**：健康期基线用 $\text{median}+k\cdot1.4826\,\text{MAD}$（$k=3$）替代 mean+kσ；
2. **自适应 PELT**：$\text{pen}=\beta_{\text{pen}}\hat\sigma_{\text{resid}}^2\ln K$，$\beta_{\text{pen}}=2$；
3. **输出完整阶段序列** $\{\hat S_b(t)\}$ 而非仅终点标签。

### 代码改动点（`code/src/q2/run_q2.py` 阶段划分函数）
- 3σ：`h_alarm = median(h_health) + 3*1.4826*MAD(h_health)`；
- PELT：`pen = 2 * np.var(hi - smooth(hi)) * np.log(len(hi))`，传入 `algo.predict(pen=pen)`；
- 落盘 `Q2_stages_{bearing}.csv` 增加逐点阶段列（不只 `Q2_summary.json` 的终点）。

### 重跑预期
长寿命轴承（Bearing3_1/3_2）出现清晰 H→D→F 三段，$t_{\text{alarm}}/T_{\text{EOL}}\le0.8$；短寿命轴承仍可能近乎直接 D→F（数据本身如此，可接受）。
